# AD 디렉션 — 메인 보드 UI 리뉴얼 가이드

> 작성: AD / 2026-04-26
> 목적: 현재 결과물(이미지 1)과 목표 샘플(이미지 2)의 격차를 좁히기 위해 작업자별 구현 사항을 분배

## 1. AD 종합 진단

### 현 상태 진단 (이미지 1)
현재 화면은 **기능적으로는 동작**하지만, AD 관점에서 다음과 같은 결정적 결함이 있다:

1. **분위기 부재** — 단색 검정 배경에 카드만 떠 있어, "포스트아포칼립틱 서울"이라는 세계관이 전혀 전달되지 않는다. 이 게임은 *Cultist Simulator*처럼 레이아웃 자체가 분위기여야 하는데, 지금은 행정 양식 같다.
2. **카드 프레임의 평면성** — 모서리 둥근 박스에 텍스트만 얹은 형태. DESIGN.md에서 명시한 "군사 매뉴얼과 한국 공공기관 서류가 뒤섞인 생존 기록" 톤이 시각적으로 구현되지 않았다.
3. **정보 위계의 부재** — 카드에 표시된 정보(아이콘, 배지, 게이지)가 부족해, 유저가 한눈에 상태를 파악할 수 없다. 샘플은 카드 한 장에 6~8개의 시각 정보를 압축해 담고 있다.
4. **헤더 부재** — 시간/계절/기온이 좌측 사이드바에 묻혀 있어, "지금 이 순간"의 긴장감이 전달되지 않는다. 샘플은 상단 중앙에 거대한 시계를 두어 게임의 호흡을 시각화했다.
5. **이중 언어(한/영) 미적용** — 이 장르 팬들이 익숙한 *Slay the Spire* 류의 군용 매뉴얼 톤에서 영문 서브타이틀은 필수 장식 요소. 현재는 한국어 단일.
6. **동료 패널의 정보량 부족** — 샘플은 초상화/나이/HP%/상태/장착/액티브 스킬까지 노출. 현재는 이름 + 호감도만.

### 우선순위
**P0 (즉시):** 배경 이미지 시스템, 카드 프레임 리뉴얼, 헤더 바
**P1 (필수):** 카드 정보 위계(배지/게이지), 사이드바 HUD 개편, 이중 언어
**P2 (마감 보강):** 동료 패널 고도화, 그레인/스캔라인 등 환경 효과

## 트랙 A — 배경/환경 분위기

### 담당 워커 프로필
**프론트엔드 + 가벼운 그래픽 감각.** CSS 합성에 능숙하고, 이미지/그라디언트로 분위기를 만들 수 있는 사람.

### 목표
검정 단색 배경을 **"폐허가 된 서울 거리"** 이미지로 교체하여, 보드 자체가 세계관의 일부가 되도록 한다. 단, 카드 가독성을 해치지 않는 선에서 **항상 어둡게**, **항상 멀리** 있어야 한다.

### 현재 상태
- `css/board.css` 또는 `css/layout.css`의 보드 컨테이너 배경이 단색 (`--bg-base: #0d0d0d`).
- DESIGN.md에 그레인 텍스처 결정은 있으나 실제 이미지 레이어는 없음.

### 구현 사항
1. **배경 이미지 레이어 추가** — `body` 또는 `#game-board` 컨테이너에 `background-image: url('assets/bg/seoul-ruins-{day|night}.webp')`. 낮/밤 두 종 준비.
2. **다크닝 오버레이** — `linear-gradient(rgba(10,10,10,0.85), rgba(10,10,10,0.7))`를 이미지 위에 겹쳐 콘텐츠 가독성 확보.
3. **컬러 틴팅** — 낮: 따뜻한 앰버 톤(`rgba(200,160,96,0.05)` 미세 오버레이), 밤: 청색 틴트(`#090d12` 베이스 + `rgba(80,144,192,0.08)`).
4. **그레인 텍스처** — DESIGN.md에 명시된 SVG 인라인 노이즈를 `::after` pseudo로 보드 전역에 깔기. `mix-blend-mode: overlay` + `opacity: 0.04`.
5. **비네팅** — 화면 가장자리를 어둡게 하는 `radial-gradient` 오버레이를 `body::before`에 추가.

### 산출물
- 배경 이미지 에셋 2종(`bg-day.webp`, `bg-night.webp`) — 1920×1080, 50KB 미만
- `css/atmosphere.css` (신규) — 모든 환경 레이어 정의
- `index.html`에 atmosphere.css 등록

### 주의
- 카드가 **반드시** 배경보다 시각적으로 우위에 있어야 한다. 배경이 튀면 게임 망친다.
- 야간 모드 토글은 트랙 D(HUD)와 협업해 `body[data-time="night"]` 데이터 속성으로 트리거.

## 트랙 B — 카드 프레임 시스템

### 담당 워커 프로필
**CSS 마스터급.** pseudo-elements, SVG, 마스킹에 익숙해야 함. UI 컴포넌트 시스템 설계 경험자.

### 목표
**모든 카드를 "군사 매뉴얼 박스"로 재설계.** 4모서리에 stencil 코너 브래킷, 좌상단/우상단에 메타 배지 영역, 하단에 정보 푸터를 가진 통일된 프레임 시스템을 만든다.

### 현재 상태
- `css/cards.css`에 카드 기본 스타일 존재 (둥근 모서리, 단색 보더).
- 코너 브래킷, 메타 배지 슬롯 없음.

### 구현 사항
1. **코너 브래킷** (4 모서리 L자 마크)
   - 방법 A: SVG 인라인을 `::before`/`::after`에 background-image로
   - 방법 B: `clip-path` + 보더 합성
   - 권장: SVG 마스크 (앰버 컬러 변경 자유롭다)
   - 크기: 카드 폭의 8~10%, 색상: `--accent-primary` 또는 `--accent-dim`
2. **카드 헤더 행** — 카드 상단 6~10px 영역에 다음 슬롯 배치:
   - 좌상단: 카테고리 아이콘 (📍 장소, ◆ 바닥, 💼 휴대 등)
   - 우상단: 속성 배지 슬롯 (⚠ 위험, ❄ 추위, ⚡ 전력 등 — 다중 가능)
3. **카드 본체** — 기존 이미지/이모지 영역, 비율 유지
4. **카드 푸터** — 하단 18~24px 행에:
   - 좌측: 주요 메트릭 (`Requirements: 12`, `2TP` 등)
   - 우측: 액션 힌트 (`탐색`, 클릭하여 사용 등)
5. **테두리 톤** — 등급별 색상 시스템(`--rarity-*`)은 유지하되, 두께 1px → 2px, 코너 브래킷이 있는 모서리는 보더 끊기 (mask로 처리)
6. **호버 상태** — 코너 브래킷이 미세하게 바깥으로 퍼지는 애니메이션 (4px translate, 120ms ease)

### 산출물
- `css/cards.css` 리팩터 (코너 브래킷 시스템 추가)
- `assets/svg/corner-bracket.svg` (4방향 단일 SVG, CSS로 회전)
- `js/render/CardFactory.js`에 `meta-icons` 슬롯 추가 (트랙 E와 협업)

### 의존
- 트랙 E(카드 정보 위계)와 **DOM 구조 합의 선행 필수**. 두 트랙 PR이 충돌하지 않도록 먼저 카드 HTML 골격 PR 한 번을 합의 후 분기.

## 트랙 C — 헤더 / 탑바

### 담당 워커 프로필
**컴포넌트 신규 구축.** DOM 구조 짜고 기존 GameState에서 데이터 바인딩 가능한 사람.

### 목표
샘플 상단 중앙의 `Day 18 | 14:32 | -4°C` 같은 **거대한 헤더**를 신규 구성. 게임의 "지금 이 순간"이 화면 상단 중앙에서 항상 시야에 들어와야 한다.

### 현재 상태
- 좌측 사이드바에 `Day 1`, `06:00`, `봄`, `12°C`가 작게 들어가 있음.
- 헤더 영역 자체가 없음 — 보드가 화면 상단부터 시작.

### 구현 사항
1. **신규 헤더 DOM** — 보드 위쪽에 `#game-header` 컨테이너 추가. 높이 56~64px.
2. **3분할 레이아웃**:
   - 좌측: 게임 로고 또는 메뉴 토글 (옵션)
   - **중앙(주인공)**: `Day {n}` | `{hh:mm}` | `{temp}°C` — 폰트 `--font-title` (Geist), 사이즈 `--font-size-xl`(28px) 또는 그 이상
   - 우측: 계절 + 날씨 아이콘 + 알림 아이콘
3. **시간/온도 색상 코딩**
   - 시간: 낮(`--accent-primary`), 밤(`--text-info`)
   - 온도: 영하(`--text-info`), 더위(`--text-warn`), 정상(`--text-primary`)
   - 구분자 `|`는 `--text-dim`로 약화
4. **반응 데이터 바인딩** — 기존 `GameState.day`, `GameState.time`, `GameState.weather.temp` 구독.
5. **레이아웃 영향** — 보드 영역의 상단 오프셋이 변경됨. 1920×1080 고정 해상도이므로, 보드 그리드의 `grid-template-rows`나 `padding-top`을 함께 조정 필요.
6. **사이드바 정리** — 좌측 사이드바의 Day/시간/온도는 헤더 이동 후 **제거**. 사이드바는 캐릭터 상태/스탯에 집중. (트랙 D 협업)

### 산출물
- `js/ui/HeaderBar.js` (신규 컴포넌트)
- `css/header.css` (신규)
- `index.html` 헤더 노드 추가

### 의존
- 트랙 D와 **사이드바에서 어떤 정보를 빼낼지** 합의 필요.

## 트랙 D — 사이드바 HUD 개편

### 담당 워커 프로필
**기존 HUD 리팩터링 + 컴포넌트 분해 경험자.** 게이지/메트릭 컴포넌트 통일 작업 가능한 사람.

### 목표
좌측 사이드바를 **"플레이어 콕핏"**으로 재설계. 샘플처럼 지도(MAP) → 상태(STATUS) → 소음(NOISE) → 무게(WEIGHT) → 퀘스트(QUESTS) 5섹션으로 명확히 분리.

### 현재 상태
- 좌측 사이드바 존재. HP/수분/영양/스태미나/피로 게이지 있음.
- 지도 섹션 있으나 라벨/마커 부족.
- 소음 미터 텍스트만 있음 (게이지 없음).
- 무게 미터 텍스트만 있음.
- 퀘스트 섹션 부재 (별도 모달로만).

### 구현 사항
1. **섹션 구획화** — 각 섹션 사이 1px `--border-dim` 구분선 + 섹션 타이틀 (`상태 (STATUS)` 형식, 트랙 G 협업)
2. **지도 섹션 강화**
   - 마커 종류: 현재 위치(▼ 청록), 발견 지점(■ 적색/녹색), 목적지(★)
   - 마커에 마우스 오버 시 툴팁
   - 미니맵 폭 풀로 사용
3. **상태 섹션 재배치**
   - 각 행: 아이콘 + 이름(한/영) + 게이지 + 수치
   - 게이지: 높이 6px, 둥근 모서리, 색상은 `--stat-*` 토큰 사용
   - 위험 임계값 도달 시 깜빡임(120ms 펄스) — DESIGN.md의 야간 플리커 패턴 응용
4. **소음 미터** — 신규 게이지 컴포넌트
   - 높이 8px, 0~100% 비율
   - 30% 미만(녹색), 30~70%(노랑), 70%+(적색, 펄스)
   - 게이지 아래 한 줄 라벨: `위험! 소음 발생` 등 상태 텍스트
5. **무게 미터** — 신규 게이지
   - 현재/최대 (kg) 숫자 표시
   - 80% 이상 시 빨간색
6. **퀘스트 섹션 신규** — 사이드바 하단에 체크리스트 형태
   - `☑ {퀘스트명} (진행중/완료)` 행 단위
   - 최대 3~5개 노출, 나머지는 `+ N more` 링크로 모달

### 산출물
- `js/ui/Sidebar.js` 리팩터
- `js/ui/components/Gauge.js` (신규 공용 컴포넌트 — 다른 트랙도 사용)
- `css/sidebar.css` 또는 기존 파일 분할

### 의존
- 트랙 C(헤더)와 시간/날씨 정보 이전 합의
- 트랙 G(이중 언어)와 라벨 데이터 합의
- 게이지 컴포넌트는 **공용**이므로 트랙 E도 사용 가능 (먼저 머지)

## 트랙 E — 카드 정보 위계

### 담당 워커 프로필
**`CardFactory.js` 깊은 이해자 + 데이터 모델 확장 경험.**

### 목표
카드 한 장에 표시되는 정보를 **샘플 수준으로 압축**한다. 샘플의 카드는 평균 6~8개의 시각 정보(이름/영문/카테고리/속성/수치/게이지/수량/장착여부)를 담고 있다.

### 현재 상태
- 카드: 이름 + 이미지 + 무게(kg) + 일부 메트릭만 표시.
- 속성 배지(⚠❄⚡), 내구도 게이지, 수량 배지, 장착 상태 배지 모두 없음.

### 구현 사항
1. **속성 배지 시스템** (카드 우상단)
   - 정의: `js/data/itemAttributes.js` 신규. 키: `cold`, `radiation`, `electric`, `fragile`, `quest`, `crafting` 등
   - 각 아이템 데이터에 `attributes: ['cold', 'fragile']` 필드 추가 (트랙 G 협업으로 데이터 확장)
   - 렌더: 아이콘 + 배경 원형, 다중일 경우 가로 스택 (최대 3개)
2. **내구도 게이지** (카드 본체 하단 또는 이미지 하단)
   - 높이 3px의 가는 바
   - 100%(녹색) → 50%(노랑) → 25%(적색)
   - 무기/도구/장비에만 노출
3. **수량 배지** (카드명 우측)
   - `x3`, `x15` 형태. `--accent-primary` 색상.
   - 1개일 때 노출 안 함 (단, 스택 가능 아이템은 항상 노출 — `stackConfig.js` 참조)
4. **장착 상태 배지** (카드 좌상단 또는 우상단)
   - `Equipped` 라벨 (앰버 박스, 1px 보더)
   - GameState에서 장착 슬롯 검사 후 적용
5. **카드 푸터 정보 통일**
   - 장소 카드: `Requirements: {n}` (탐색 비용 또는 잠금 조건)
   - 아이템 카드: 무게 + 카테고리 약자 (`MEDICAL`, `FOOD`, `MELEE` 등)
   - 좌하단 정렬, `--text-secondary` 색상, `--font-size-xs`
6. **카드 카테고리 라벨** (카드 좌상단 캡)
   - 샘플에 `MEDICAL`, `DRINK`, `FOOD`, `MELEE` 등이 카드 상단에 캡 형태로 박혀 있음
   - 배경: `--accent-dim`, 텍스트: `--bg-base`, 작은 캡슐 형태

### 산출물
- `js/render/CardFactory.js` 확장
- `js/data/itemAttributes.js` 신규
- `css/cards.css`에 배지/게이지 스타일 추가
- `js/data/items_*.js` 일괄 업데이트 (attributes 필드 — 트랙 G와 동시 진행 가능)

### 의존
- 트랙 B와 카드 DOM 골격 합의 (선행)
- 트랙 D의 게이지 컴포넌트 재사용
- 트랙 G의 카테고리 영문명 데이터

## 트랙 F — 동료 패널 고도화

### 담당 워커 프로필
**NPC 시스템 이해자.** `NPCSystem`, `EquipmentModal` 코드 익숙한 사람 우선.

### 목표
우측의 빈약한 동료 카드를 샘플 수준의 **"동료 콕핏"**으로 확장. 초상화/나이/HP%/상태/장착 슬롯/액티브 스킬을 한 패널에 노출.

### 현재 상태
- `동반자` 패널: 이름 + 호감도 별 + 상태 텍스트 + 위치 정도.
- 초상화, 장착 시각화, 액티브 스킬 표시 없음.

### 구현 사항
1. **초상화 영역** — 상단에 정사각형 영역 (160×160px 정도). 이미지 또는 SVG 일러스트.
   - 데이터: `npcs.js`에 `portrait` 필드 추가
   - 부재 시 이니셜 + 색상 배경 fallback
2. **헤더 정보** — 이름(한) + `(영문명)`, `{나이}세`, `HP {퍼센트}%`
3. **상태 행** — `상태: 양호 / 안정 / 부상 / 감염` 등. 색상 코딩 (`--text-good`, `--text-warn`, `--text-danger`).
4. **장착 슬롯 시각화** — 3~4개의 정사각 슬롯 (무기/방어구/소모품).
   - 슬롯에 아이템 아이콘 + 수량
   - 빈 슬롯은 `--slot-empty` + 점선 보더
5. **액티브 스킬** — 별도 행. 스킬 아이콘 + 이름 + (대기중 / 사용가능) 상태.
6. **trust 시스템 연계** — 호감도 별은 유지하되 패널 푸터로 이동, 메인 영역은 전투/생존 정보 우선.

### 산출물
- `js/ui/CompanionPanel.js` 또는 `js/ui/NPCPanel.js` 리팩터
- `css/npc-panel.css` 확장
- `js/data/npcs.js` 데이터 필드 추가 (`portrait`, `age`, `englishName`, `activeSkill`)
- 초상화 에셋 — 동료 캐릭터별 1종

### 의존
- 트랙 G(이중 언어)의 영문명 데이터 합의
- CLAUDE.md의 동반자 능력 ×1.0~×1.3 trust 스케일링 시스템 유지

## 트랙 G — 이중 언어 (한/영) 시스템

### 담당 워커 프로필
**i18n 경량 도입 + 기존 데이터 일괄 업데이트 가능한 사람.** 데이터 검증 스크립트 작성 경험자 우대.

### 목표
샘플처럼 **모든 라벨/카드명/섹션 헤더에 영문 서브타이틀** 추가. 별도 언어 전환은 아니고, **항상 같이 노출**되는 장식적 i18n.

### 현재 상태
- 모든 텍스트 한국어 단일.
- 영문 데이터 필드 없음.

### 구현 사항
1. **데이터 필드 일괄 추가**
   - 아이템: `nameEn` 필드 → `items_*.js` 7개 파일 모두
   - 장소/지구: `nameEn` → `districts.js`
   - NPC: `nameEn` → `npcs.js`
   - UI 라벨: `js/i18n/labels.js` 신규 (사이드바 섹션, 헤더, 버튼 등)
2. **표시 규칙**
   - 카드명: 한글 큰 글씨 + 영문 작은 글씨 (1줄 또는 2줄)
   - 섹션 헤더: `장소 (LOCATIONS)` 형태
   - UI 버튼: 한글만 또는 한글 + 영문 약자 (작업자 재량)
3. **렌더 헬퍼** — `js/i18n/bilingualLabel(item) => "{name}\n{nameEn}"` 형태 유틸
4. **검증 스크립트 확장** — 기존 `js/data/validate.js`에 "모든 아이템에 nameEn 존재" 체크 추가

### 산출물
- `js/i18n/labels.js` (신규)
- 모든 데이터 파일에 `nameEn` 추가
- `js/data/validate.js` 확장
- `js/render/CardFactory.js`에서 bilingualLabel 사용으로 교체

### 의존
- 다른 모든 트랙이 라벨 표시할 때 이 시스템을 사용해야 함 → **G가 가장 먼저 머지되어야 효율적**.
- CLAUDE.md의 검증 명령 (`node --input-type=module js/data/validate.js`) 통과 확인 필수.

## 4. 공통 규칙 (모든 트랙 적용)

1. **DESIGN.md 토큰만 사용**, 색상/폰트/간격 하드코딩 금지. 새 토큰이 필요하면 `variables.css`에 먼저 추가하고 PR.

6. **QA 모드** — 모든 PR은 `/qa` 통과 + `DESIGN.md` 토큰 위반 검수.
