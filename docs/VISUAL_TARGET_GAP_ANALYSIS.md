# 시각 타겟 격차 분석 — 가상 이미지 vs 실제 게임

> 작성: AD / 2026-04-28
> 대상: `시작화면` / `캐릭터선택화면` / `메인플레이화면` 3 화면
> 입력: 사용자 제공 게임이미지.zip (실제 PNG 3 + 가상 JPG 3)
> 기준 문서: `~/.claude/Instructions/briefs/AD_GUIDE_UI_REVAMP.md` (트랙 A-G), `DESIGN.md`

---

## TL;DR

3 화면 모두 분석한 결과, **격차의 ~80%는 기존 `AD_GUIDE_UI_REVAMP.md`의 7개 트랙(A-G)으로 커버됨.** 트랙들이 이미 이 가상 이미지를 목표로 설계되어 있다. 미커버 격차는 다음 3가지 — 신규 트랙 또는 기존 트랙 확장으로 처리:
- **시작화면 종이/문서 컬라주 톤** (트랙 A 확장)
- **캐릭터선택 풀-바디 일러스트 + 가로 게이지** (트랙 F 확장 또는 신규 트랙 I)
- **사이드바 좌측 정렬 사실상 처음부터 재구성** (트랙 D 더 큰 범위)

## 1. 시작화면 격차

### 실제 게임 (현재 — `js/screens/MainMenu.js` 170줄)

- 풀 화면 영웅 일러스트 (포스트아포칼립틱 여성 + 좀비 무리)
- 타이틀: 영문 `RUINED CITY / CARD SURVIVAL · 2099` 중앙
- 슬롯 카드 3개 (Day 1, +, +) 가로 배치
- 멘탈 컬렉션 표시 (2/24)
- 설정 버튼

### 가상 이미지 (목표)

- **종이/서류 베이스 배경** — 공공기관 문서, 신문지, 통제 테이프 컬라주
- 한글 타이틀 강조: `Card Survival: Ruined City`
- 부제: `2026년 1월, 서울이 무너졌다.`
- 메뉴 4 버튼 세로: `새 게임 / 이어하기 / 엔딩 갤러리 / 설정`
- 좌상단: TV 뉴스 클립 (서울역 강제 통금)
- 우상단: 보라매병원 의사 등재 신문
- 황색 통제 테이프 (`통제구역`) 하단 가로 띠
- 광화문/한강대교 잔해 사진 컬라주

### 격차 매트릭스

| 항목 | 실제 | 목표 | AD 트랙 |
|------|------|------|---------|
| 분위기 | 영웅 일러스트 | 서류 컬라주 | **A 확장** (기존 트랙 A는 게임 보드 배경 위주) |
| 타이틀 | 영문 위주 | 한글 위주 + 영문 부제 | G (이중 언어) |
| 메뉴 구조 | 슬롯 3개 | 4 버튼 (이어하기 / 엔딩갤러리 분리) | **신규** (메뉴 정보 구조 변경) |
| 디제틱 디테일 | 없음 | 신문 / 통제 테이프 / 뉴스 영상 | **A 확장** |

### 작업량 추정
- HTML 재구성: `screen-main-menu` 컨테이너 안 레이아웃 1회 재작성 (~150 LoC)
- CSS 신규: `css/start-screen.css` 또는 `screens-menu.css` 안에 신규 패턴 (~200 LoC)
- 에셋: 신문 컬라주 PNG 1-2종 (외주 또는 나노 바나나)

## 2. 캐릭터선택화면 격차

### 실제 게임 (현재 — `js/screens/CharCreate.js` 597줄)

- 검정 단색 배경
- "생존자 생성" 타이틀
- 6개 직업 카드 그리드 (이지수/강민준/박영철/최형식/윤재혁/정대한)
- 선택된 직업: 이름/직업/HP/체력/인내심/스테미나/무게 색 배지 + 텍스트 박스
- 시작 위치 텍스트
- 하드코어 모드 안내
- "생존 시작" / "메인 메뉴" 버튼

### 가상 이미지 (목표)

- **폐허 도시 풍경 배경** (눈 내리는 한강대교)
- 좌측: **풀-바디 캐릭터 일러스트** (의사 이민우, 의료형) — 거의 1:1 비율 큰 영역
- 중앙 상단: 개인 정보 (나이 38 / 배경 응급의학과 의사)
- 중앙 중간: **능력치 가로 게이지** 5종 (정신건강/스테미나/의료스킬/전투/소등기)
- 우측 상단: 강점 / 약점 (각 4개 항목 리스트)
- 우측 하단: 개인 목표 / 출발지
- 하단: 6개 직업 작은 카드 (1번 활성, 6개 번호) + 미니 능력치 바
- "캐릭터 선택" + "서울 서바이벌" 타이틀

### 격차 매트릭스

| 항목 | 실제 | 목표 | AD 트랙 |
|------|------|------|---------|
| 배경 | 검정 단색 | 폐허 풍경 (눈) | A (배경) |
| 캐릭터 시각 | 텍스트만 | 풀-바디 일러스트 좌측 큰 영역 | **F 확장** 또는 **신규 트랙 I** |
| 능력치 표현 | 색 배지 텍스트 | 가로 게이지 5개 | D (Gauge.js 재활용) |
| 스토리/배경 | 긴 단락 텍스트 | 강점/약점 리스트 + 개인 목표 + 출발지 | **데이터 모델 확장** |
| 직업 카드 | 6 카드 그리드 (선택+상세) | 좌+중+우 3분할 + 하단 카드 | **레이아웃 재구성** |

### 작업량 추정
- 데이터: `npcs.js` 또는 `characters.js`에 `portrait`, `englishName`, `strengths[]`, `weaknesses[]`, `personalGoal`, `startLocation` 필드 추가 (~100 LoC 데이터)

- HTML/CSS: `screen-char-create` 안 레이아웃 + 게이지 컴포넌트 (~250 LoC)
- 의존: 트랙 D 게이지 컴포넌트 + 트랙 G 영문 데이터

## 3. 메인플레이화면 격차

### 실제 게임 (현재 — `js/screens/Basecamp.js` 434줄, `css/board.css` 291줄)

- 검정 단색 배경
- 좌측 사이드바: Day/시간/계절/온도, 캐릭터 정보 (간단)
- 보드 3행: 장소(top) / 바닥(middle) / 휴대(bottom)
- 카드: 둥근 박스 + 이름 + 이미지 + 무게(kg)
- 우측 패널: 거의 비어있음

### 가상 이미지 (목표)

- 폐허 도시 배경 (보드 뒤 비침)
- **헤더 상단 중앙**: `Day 18 | 14:32 | -4°C` 거대 폰트
- 좌측 사이드바 5섹션:
  - 지도 (MAP) - 미니맵 + 마커
  - 상태 (STATUS) - HP/배고픔/갈증 게이지
  - 소음 미터 (NOISE METER) - 가로 게이지
  - 휴대 무게 (WEIGHT) - 게이지 + 정상/과중
  - 퀘스트 (QUESTS) - 체크리스트
- 보드 3행: 카드에 **코너 브래킷** + **카테고리 캡** + **속성 배지** + **푸터 정보**
- 카드 상단 캡: `MEDICAL`, `FOOD`, `MELEE` 등 영문 약자
- 모든 라벨 한/영 병기 (`장소 (LOCATIONS)`)
- 우측: **동료 패널** (초상화 + 나이 + HP% + 상태 + 장착 슬롯 + Active Skill)

### 격차 매트릭스 — AD_GUIDE_UI_REVAMP.md 매핑

| 항목 | 실제 | 목표 | AD 트랙 (이미 정의됨) |
|------|------|------|----------------------|
| 배경 | 검정 단색 | 폐허 도시 (어두운 오버레이) | **A** ✓ |
| 헤더 | 좌측 사이드바 안에 묻힘 | 상단 중앙 거대 헤더 | **C** ✓ |
| 사이드바 | 캐릭터 정보 위주 | 5 섹션 콕핏 | **D** ✓ |
| 카드 프레임 | 평면 박스 | 코너 브래킷 + 캡 라벨 | **B** ✓ |
| 카드 정보 | 이름+이미지+무게 | 6-8 시각 정보 | **E** ✓ |

## 5. 신규 트랙 I 제안 — 시작/캐릭터 화면 컬라주 톤

### 담당 워커 프로필
**일러스트 + CSS 합성 능력자.** 신문/문서 톤 컬라주 레이아웃 경험자.

### 목표
시작화면과 캐릭터선택화면을 **단순 일러스트 + 텍스트 박스에서 → 디제틱 서류 컬라주**로 전환. 영웅 캐릭터 일러스트는 *유지하되* 위계를 낮추고, 신문/통제 테이프/뉴스 클립이 분위기를 만든다.

### 시작화면 변경 사항
1. **풀 화면 캐릭터 일러스트 → 배경으로 후퇴 + 문서 레이어**
   - 캐릭터 일러스트는 opacity 0.5 + 어두운 그래디언트로 깔리고
   - 위에 신문 클리핑 + 통제 테이프 + 메뉴 4 버튼 배치
2. **메뉴 정보 구조 변경**
   - 슬롯 3개 → 4 버튼 (`새 게임` / `이어하기` / `엔딩 갤러리` / `설정`)
   - "이어하기" 클릭 시 슬롯 선택 모달
3. **에셋 신규**
   - `assets/images/menu/newspaper-clipping-news.png` (TV 뉴스 클립)
   - `assets/images/menu/newspaper-clipping-doctor.png` (의사 등재)
   - `assets/images/menu/control-tape.png` (반복 패턴)
   - `assets/images/menu/ruined-buildings.png` (광화문/한강 잔해)

### 캐릭터선택 변경 사항
1. **풀-바디 캐릭터 일러스트 좌측 50%** 영역 (현재 텍스트 박스 위치 대체)
2. **중앙 30%**: 능력치 가로 게이지 5개 + 강점/약점 리스트
3. **하단 20%**: 6 직업 작은 카드 (번호 1-6 + 초상화 + 미니 게이지)
4. **데이터 확장**:
   - 직업별 `portrait` (풀-바디) + `portraitSmall` (초상화)
   - `englishLabel` (의료형/전투형/제작형 등)
   - `strengths[]` / `weaknesses[]` 각 3-4개
   - `personalGoal`, `startLocation`

### 작업량
- HTML/CSS: ~400 LoC (시작 + 캐릭터선택 합산)
- 데이터: ~100 LoC
- 에셋: 6 캐릭터 풀-바디 + 4-6 신문 클리핑

### 의존
- 트랙 G (영문 라벨)
- 트랙 D (게이지 컴포넌트)
- 트랙 F (동료 패널 풀-바디 패턴)

### 복잡도
**HIGH** — 데이터 모델 확장 + 레이아웃 2 화면 재작성 + 에셋 다수.
**CC 시간**: 6-10 시간 + 외주 일러스트 대기.

## 6. 권장 우선순위 + 구현 순서

```
Phase A — 메인 보드 (AD_GUIDE_UI_REVAMP 트랙)
  G+ (이중언어) → CST 이미 진행 ✓
  D (사이드바 HUD) → 게이지 컴포넌트 우선
  C (헤더) → 사이드바와 협업
  B (카드 프레임) + E (카드 정보) → 합의 PR 후 병렬
  A (배경) → 보드 분위기
  F (동료 패널) → 후순위

Phase B — 시작/캐릭터 화면 (신규 트랙 I)
  I-1 (시작화면 컬라주) → 데이터 영향 적음, 먼저 작업 가능
  I-2 (캐릭터선택 일러스트 + 게이지) → D 게이지 + 데이터 확장 후
```

> **현실적 제안**: 메인 보드 (Phase A)가 더 자주 보이는 화면 → 우선. 시작/캐릭터선택은 1회 본 후 게임 안 들어가면 안 보임 → 후순위.

## 8. 미해결 사항 (사용자 결정 필요)

1. **에셋 외주 vs 자체 생성**
   - 신문 클리핑, 통제 테이프, 폐허 사진은 나노 바나나로 가능
   - 6 캐릭터 풀-바디 일러스트는 **일관성**이 중요 → 외주 또는 단일 모델로 batch 생성 권장
2. **메뉴 구조 변경 (시작화면)**
   - 현재 슬롯 시스템 → 4 버튼으로 전환 시 기존 저장 슬롯 호환성 검토 필요
   - "이어하기" 모달 안에 슬롯 선택 UI 신규
3. **캐릭터 데이터 모델 확장 시점**
   - 신규 필드(`portrait`, `strengths`, `weaknesses`)를 trait/이벤트 시스템과 연결할지
   - 단순 표시용으로 시작하고 추후 게임플레이 영향 검토할지
4. **트랙 I-2 캐릭터 능력치 표시**
   - 가상은 5종 (정신건강/스테미나/의료스킬/전투/소등기)
   - 실제 GameState 스탯과 1:1 매핑인지 직업 특성으로 별도 표시인지

## 9. 결론

격차의 80%는 이미 식별된 AD 트랙으로 처리 가능. **메인 보드는 우선순위 진행**, **시작/캐릭터선택은 신규 트랙 I로 분리**. 신규 작업의 핵심 제약은 **에셋 (6 풀-바디 일러스트)** — 워커 분배 전 일러스트 외주 계약이 선행 필요.

작업 진입 시작점은 **트랙 D 게이지 컴포넌트** — 사이드바와 캐릭터선택 양쪽에서 재사용. 한 번 만들면 두 화면 모두 추진력 확보.
